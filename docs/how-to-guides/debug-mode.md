# デバッグモードの使い方

ゲームの挙動がおかしい、スプライトが表示されない、動作が重い...そんなときはデバッグモードを使いましょう。画面左上にリアルタイム情報が表示され、問題の原因を素早く絞り込めます。

## デバッグモードの有効化

### 方法1: init.aria で起動時から有効にする

`init.aria` に以下を追加すると、ゲーム起動時からデバッグ表示が有効になります。

```aria
debug on
```

`debug off` で無効化もできます。リリース版では通常 `off` にしておきます。

### 方法2: 実行中に F3 キーで切り替える

ゲーム実行中に **F3 キー** を押すと、デバッグ表示の ON/OFF を切り替えられます。スクリプトを修正せずに一時的に確認したい場合に便利です。

## デバッグ表示の内容

デバッグモード有効時、画面左上に以下の情報が緑色と黄色で表示されます。

| 表示項目 | 色 | 意味 |
|---------|-----|------|
| FPS | 緑 | 現在のフレームレート。60未満なら描画負荷が高い |
| PC | 緑 | Program Counter。現在実行中の命令インデックス |
| Sprites | 緑 | 現在登録されているスプライト総数 |
| Draw Calls | 黄 | このフレームでの描画回数 |
| Tex Loads | 黄 | テクスチャ読み込み回数（累計） |
| Color Cache | 水色 | 色情報のキャッシュ件数 |
| Tex Cache | 水色 | テクスチャのキャッシュ件数 |
| Tex Stats | 水色 | テクスチャキャッシュの使用率（%） |

### 各項目の見方

**FPS**
: 60fps を安定して維持できているか確認します。30fps を切るようなら、スプライト数の削減や画像サイズの見直しを検討してください。

**PC**
: スクリプトのどの行相当を実行しているか示します。フリーズした場合、この値が動いていればVMは稼働中。動かなければ無限ループや待機状態の可能性があります。

**Sprites**
: 画面上に見える見えない関わらず、登録されているスプライトの総数です。予想より多い場合、`csp` で不要なスプライトを削除し忘れている可能性があります。

**Draw Calls / Tex Loads**
: 描画回数とテクスチャ読み込み回数の累計です。Tex Loads が異常に増えている場合、キャッシュが効いていない（同じ画像を繰り返し読んでいる）可能性があります。

**Tex Stats**
: テクスチャキャッシュの使用率です。100%に近いとキャッシュが一杯で、古いテクスチャが破棄され新規読み込みが発生しています。高解像度画像を多用する場合、この値が高くなりがちです。

## よくあるデバッグ作業

### スプライトが表示されない

1. F3 を押してデバッグ表示を ON
2. Sprites の数を確認。0なら `lsp` や `bg` が実行されていない
3. Sprites に登録されているのに見えない場合は `vsp` で非表示になっていないか確認
4. Z座標が他のスプライトより手前にあるか確認（Z値が大きいほど手前）

### 動作がカクつく

1. FPS が 60 を下回っているか確認
2. Sprites が極端に多い（100以上）場合は不要なスプライトを `csp` で削除
3. Tex Stats が 90% 以上なら、使用画像を減らすか解像度を下げる
4. Draw Calls がフレームごとに大きく増える場合、アニメーションやエフェクトの見直しを

### スクリプトが進まない

1. PC の値を監視。クリックしても変わらない場合は待機状態
2. `wait` や `@`、ボタン待ちで停止している可能性。スクリプトの該当位置を確認
3. PC が増え続けるのに画面が変わらない場合、表示系命令がスキップされているか、画面外に描画されている可能性があります

### メモリ使用量が気になる

1. Tex Cache の件数と使用率を確認
2. シーン切り替え時に Sprites が減っているか確認（`csp -1` で全削除されるか）
3. Tex Loads がシーンごとに大きく増える場合、キャッシュサイズ対画像サイズの見直しを

## ヒント

- デバッグ表示はゲーム画面に重なるため、スクリーンショット撮影時は F3 で OFF にしてください
- `debug on` を init.aria に書くと常時有効になります。開発中は ON、リリース直前は OFF を推奨
- F3 による切り替えは即座に反映されます。ゲームを再起動する必要はありません
