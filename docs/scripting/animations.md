# アニメーション

このドキュメントでは、AriaEngineのアニメーション機能について説明します。

## アニメーションの基本

### Tweenアニメーション

AriaEngineはTween（トゥイーン）アニメーションをサポートしており、スプライトのプロパティを滑らかに変化させることができます。

### アニメーションの種類

1. **移動アニメーション** - 位置の変化
2. **透明度アニメーション** - フェードイン/フェードアウト
3. **スケールアニメーション** - 拡大/縮小
4. **回転アニメーション** - 回転

## 位置アニメーション

### 基本的な移動アニメーション

```aria
lsp 10, "character.png", 0, 0

; X=100, Y=200へ1秒かけて移動
amsp 10, 100, 200, 1000
```

### 緩やかな移動

```aria
; 2秒かけてゆっくり移動
amsp 10, 500, 300, 2000
```

### 連続移動

```aria
; 第一段階
amsp 10, 100, 200, 500
await

; 第二段階
amsp 10, 200, 400, 500
await

; 第三段階
amsp 10, 300, 600, 500
await
```

## 透明度アニメーション

### フェードイン

```aria
lsp 10, "bg.png", 0, 0
sp_alpha 10, 0  ; 最初は透明

; 1秒かけてフェードイン
afade 10, 255, 1000
```

### フェードアウト

```aria
sp_alpha 10, 255  ; 最初は不透明

; 0.5秒かけてフェードアウト
afade 10, 0, 500
```

### 連続フェード

```aria
; フェードイン
afade 10, 255, 1000
await

; 待機
wait 1000

; フェードアウト
afade 10, 0, 1000
```

## スケールアニメーション

### 拡大

```aria
lsp 10, "item.png", 640, 360
sp_scale 10, 0.1  ; 最初は小さい

; 1秒かけて2倍に拡大
ascale 10, 2.0, 1000
```

### 縮小

```aria
sp_scale 10, 2.0  ; 最初は大きい

; 0.5秒かけて元のサイズに
ascale 10, 1.0, 500
```

### 脈動エフェクト

```aria
; 拡大
ascale 10, 1.2, 500
await

; 縮小
ascale 10, 1.0, 500
await

; ループ
goto *pulse

*pulse
    ascale 10, 1.2, 500
    await
    ascale 10, 1.0, 500
    await
    goto *pulse
```

## 回転アニメーション

### 基本的な回転

```aria
lsp 10, "arrow.png", 640, 360
sp_rotation 10, 0  ; 最初は0度

; 1秒かけて45度回転
arotation 10, 45, 1000
```

### 連続回転

```aria
; 90度回転
arotation 10, 90, 1000
await

; 180度回転
arotation 10, 180, 1000
await

; 270度回転
arotation 10, 270, 1000
await

; 360度（1周）回転
arotation 10, 360, 1000
```

## イージング関数

### イージングの種類

アニメーションにはイージング関数を適用して、動きを自然にすることができます。

```aria
; 線形（デフォルト）
ease 10, "linear"
amsp 10, 100, 200, 1000

; イーズイン（ゆっくり始まる）
ease 10, "easein"
amsp 10, 100, 200, 1000

; イーズアウト（ゆっくり終わる）
ease 10, "easeout"
amsp 10, 100, 200, 1000

; イーズインアウト（両端でゆっくり）
ease 10, "easeinout"
amsp 10, 100, 200, 1000
```

### イージングの比較

```aria
; 線形移動
lsp 10, "obj1.png", 0, 100
ease 10, "linear"
amsp 10, 1000, 100, 2000

; イーズインアウト移動
lsp 20, "obj2.png", 0, 200
ease 20, "easeinout"
amsp 20, 1000, 200, 2000
```

## アニメーションの待機

### awaitコマンド

`await`コマンドを使用して、アニメーションが完了するのを待ちます。

```aria
; アニメーション開始
amsp 10, 100, 200, 1000

; 完了を待機
await

; 次の処理へ
text "移動完了！"
```

### 複数アニメーションの待機

```aria
; 複数のアニメーションを同時に開始
amsp 10, 100, 200, 1000
afade 20, 255, 1000
ascale 30, 2.0, 1000

; 全てのアニメーションが完了するのを待機
await

text "全てのアニメーション完了！"
```

### タイムアウト付き待機

```aria
; アニメーション開始
amsp 10, 100, 200, 1000

; 最大1.5秒待機（アニメーションは1秒で完了）
await
wait 500  ; 余分な待機
```

## 並列アニメーション

### 複数スプライトの同時アニメーション

```aria
; キャラクター移動
lsp 10, "hero.png", 0, 100
amsp 10, 800, 100, 1000

; 背景フェードイン
lsp 20, "bg.png", 0, 0
afade 20, 255, 1000

; 両方のアニメーションを待機
await
```

### 複数プロパティの同時アニメーション

```aria
lsp 10, "effect.png", 640, 360

; 移動しながらフェードイン
amsp 10, 700, 400, 1000
afade 10, 255, 1000
ascale 10, 1.5, 1000

await
```

## 連続アニメーション

### 順次実行

```aria
*animation_sequence
    ; 第一段階
    amsp 10, 100, 200, 500
    await

    ; 第二段階
    amsp 10, 200, 400, 500
    await

    ; 第三段階
    amsp 10, 300, 600, 500
    await
```

### 条件付きアニメーション

```aria
let %mode, 1

if %mode == 1
    amsp 10, 100, 200, 500
    await
elseif %mode == 2
    amsp 10, 200, 400, 500
    await
endif
```

## 実践的な例

### タイトル画面のアニメーション

```aria
*title_animation
    ; 背景フェードイン
    lsp 10, "bg/title.png", 0, 0
    afade 10, 255, 1000

    ; ロゴフェードイン
    lsp 20, "logo.png", 640, 200
    sp_alpha 20, 0
    afade 20, 255, 800

    ; ボタンスケールアニメーション
    lsp_rect 30, 440, 300, 400, 60
    sp_scale 30, 0.5
    ascale 30, 1.0, 600

    await
    wait 500

    ; ボタンを有効化
    sp_isbutton 30, true
    spbtn 30, 1

    btnwait %result
    if %result == 1
        goto *game_start
    endif
```

### キャラクター登場アニメーション

```aria
*character_enter
    ; キャラクターを画面外に配置
    lsp 10, "hero.png", 1400, 100
    sp_alpha 10, 255

    ; 画面中央にスライドイン
    amsp 10, 800, 100, 800
    ease 10, "easeout"

    await

    ; 会話開始
    ミオ「こんにちは、ようこそ！」
```

### ステータス画面のエフェクト

```aria
*status_effect
    ; ステータスウィンドウ表示
    lsp_rect 10, 100, 100, 1080, 520
    sp_fill 10, "#2a2a3e", 200
    sp_round 10, 10
    sp_border 10, "#4a4a6e", 2
    sp_alpha 10, 0

    ; フェードイン
    afade 10, 255, 300
    await

    ; ステータス情報表示
    lsp_text 11, "HP: 100/100", 200, 150
    sp_alpha 11, 0
    afade 11, 255, 200
    await

    lsp_text 12, "MP: 50/50", 200, 200
    sp_alpha 12, 0
    afade 12, 255, 200
    await
```

### ダメージエフェクト

```aria
*damage_effect
    ; 被弾エフェクト
    lsp 10, "flash.png", 640, 360
    sp_alpha 10, 255
    sp_scale 10, 0.5

    ; 拡大しながらフェードアウト
    ascale 10, 2.0, 300
    afade 10, 0, 300
    await

    ; キャラクターの揺れ
    amsp 20, 780, 100, 50
    ease 20, "easeout"
    await

    amsp 20, 800, 100, 100
    ease 20, "easeinout"
    await
```

## アニメーションのキャンセル

### アニメーション中の割り込み

```aria
; アニメーション開始
amsp 10, 100, 200, 2000

; ユーザー入力でキャンセル
wait_click

; 新しいアニメーションを開始（古いアニメーションは上書き）
amsp 10, 500, 300, 500
```

## ベストプラクティス

1. **awaitを忘れない**: アニメーション完了後に処理を続行する場合、必ず`await`を使用
2. **イージングを活用**: `easeout`や`easeinout`で自然な動きを実現
3. **パラレルアニメーション**: 同時に実行できるアニメーションは並列化して効率化
4. **タイミングを調整**: アニメーションの時間はユーザー体験に合わせて調整
5. **適度なエフェクト**: 過度なアニメーションはユーザーの注意を散らす

## トラブルシューティング

### アニメーションが実行されない

1. `await`を忘れていないか確認
2. スプライトIDが正しいか確認
3. アニメーションコマンドの引数が正しいか確認

### アニメーションがカクつく

1. イージング関数を`easeinout`に変更
2. アニメーション時間を長くする
3. フレームレートを確認（F3キーでデバッグモード）

### アニメーションが期待通りに動かない

1. 複数のアニメーションが競合していないか確認
2. スプライトの初期状態を確認
3. Zオーダーが適切か確認
