# Aria コア仕様（ドラフト）

この文書は、Ariaのコア文法と命令設計の最小仕様を定義します。

## 1. 文法スタイル

1. 基本は命令形式: `op arg1, arg2, arg3`
2. 1行1文を原則とする
3. 文字列は引用符で囲む
4. ラベルは `*label`
5. コメントは `;` 以降

## 2. データモデル

1. 数値レジスタ: `%name` または `%0` 形式
2. 文字列レジスタ: `$name`
3. スプライト: `id` で一意管理
4. 実行状態: `Running / Waiting / Ended` を明示管理

### レジスタ保存範囲

数値レジスタは、名前で保存範囲を判断します。

1. `%0-%99`: 揮発。セーブにも永続にも保存しない
2. `%100-%199`: 永続。`persistent.ariasav` に保存
3. `%300-%499`: 永続。章解放やCG解放向け
4. `%500-`: セーブスロット別。`slot_XX.ariasav` に保存
5. `%p.*`, `%sys.*`, `%global.*`: 永続
6. `%v.*`, `%tmp.*`: 揮発
7. その他の名前付きレジスタ: セーブスロット別

新規スクリプトでは、意図が明確な `set_pflag/set_sflag/set_vflag` を優先します。

フラグ取得は、結果レジスタを第2引数で指定できます。

```aria
get_pflag chapter.day1 %10
get_sflag route.heroine %route
get_vflag ui.open %v.open
```

## 3. 制御構文

1. `if / else / endif`
2. `for / next`
3. `gosub / return`
4. `goto *label`

条件式で扱う演算子:

1. `== != > < >= <=`
2. `&& ||`

## 4. 描画とUI

Ariaコアは「UI自動生成」を行いません。UIは描画プリミティブ命令で構築します。

主要プリミティブ:

1. `lsp`（画像）
2. `lsp_rect`（矩形）
3. `lsp_text`（文字）
4. `sp_*`（装飾・見た目変更）
5. `btn / spbtn / btnwait`（入力）

補助命令:

1. `ui_theme "clean|classic"`（UIトークンの一括適用）

## 5. 命令責務

命令は単機能とし、次を禁止します。

1. 描画命令が入力待機を暗黙で開始する
2. テキスト命令が背景UIを暗黙で生成する
3. 待機命令が状態を書き換える以外の副作用を持つ

## 6. 互換レイヤー

既存資産互換のために高レベル命令を持つ場合、以下を満たします。

1. 互換挙動は明示設定で有効化する
2. コア挙動を上書きしない
3. 将来的にスクリプトライブラリへ置換可能な構造にする

現在の互換スイッチ:

1. `compat_mode on` で旧来の自動UI挙動を許可
2. `compat_mode off` で自動UI生成を無効化
3. 新規プロジェクトの既定は `compat_mode off`

## 7. 入力待機の結果受け取り

`btnwait` は結果レジスタを明示できます。

```aria
btnwait %0
btnwait %result
```

結果は指定レジスタに格納され、互換のため `%0` と `%r0` にも同値が入ります。

## 8. エラー方針

1. 引数不足は実行時エラー
2. 型不正は警告ではなくエラーを優先
3. 未定義ラベルは即時エラー
4. 暗黙補完は最小限にする

## 9. 命令登録

Parserは命令表を直接持たず、`CommandRegistry` から命令を解決します。

1. 命令には正規名、エイリアス、カテゴリを持たせる
2. ASM風エイリアス: `jmp/call/ret`
3. BASIC風エイリアス: `goto/gosub/return`
4. 互換命令は `Compatibility` カテゴリへ隔離する
5. 新命令追加時はParserではなくRegistryに登録する
