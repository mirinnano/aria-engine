# Aria 言語理念

この文書は、Aria言語の設計判断を統一するための「理念」を定義します。

## 1. 言語の役割

Ariaは、次の性質を持つスクリプト言語を目指します。

1. ASMの良さ: 命令が小さく単機能で、実行結果を予測しやすいこと
2. BASICの良さ: ラベル、分岐、サブルーチンを読みやすく書けること
3. C++の良さ: 分割、名前管理、明示的な保存範囲で大規模化に耐えること

## 2. コア方針

1. コアは最小限
2. 高度機能はライブラリ化
3. UIはスクリプトで構築
4. 1命令1責務
5. 暗黙副作用の禁止
6. 未定義動作の禁止

## 3. 非目標

次はコア言語の責務に含めません。

1. 特定のUIレイアウトを自動生成すること
2. 作品固有の演出や画面遷移を自動提供すること
3. 高レベル機能を命令にハードコードすること

## 4. 設計ルール

1. テキスト命令は文字列状態の更新のみを行う
2. 待機命令は待機状態の遷移のみを行う
3. 選択命令は選択状態の管理のみを行う
4. 描画は常に描画プリミティブ命令で行う
5. 互換レイヤーとコアは分離する

## 5. 命令セットの層

Aria命令は、内部的にカテゴリを持ちます。

1. `Core`: `mov/add/cmp/jmp/call/ret` などの低レベル実行命令
2. `Script`: `defsub/getparam/script/include/alias` などの構造化補助
3. `Render`: `lsp/lsp_rect/lsp_text/sp_*` などの描画プリミティブ
4. `Text`: `textclear/waitclick/backlog/kidokumode` などの文章制御
5. `Input`: `btn/spbtn/btnwait/rmenu` などの入力制御
6. `Audio`: BGM/SE/MP3/DWAVE
7. `Save`: セーブ/ロードと保存許可
8. `Flags`: 永続/セーブ別/揮発フラグ
9. `System`: ウィンドウ、タイトル、終了、デバッグ
10. `Compatibility`: NScripter風または旧Aria互換の高レベル命令

新命令は、必ずどのカテゴリに属するかを決めてから追加します。

## 6. 保存範囲

フラグとレジスタは、次の3種類を明示的に分けます。

1. `persistent`: 全セーブ共通。設定、既読、章解放、CG解放
2. `save`: セーブスロット別。進行、分岐、好感度
3. `volatile`: 起動中だけ。一時UI、ボタン結果、作業値

`set_pflag/set_sflag/set_vflag` はこの区分を直接表します。
`%` レジスタは互換性のため番号帯でも分類します。

1. `%0-%99`: volatile
2. `%100-%199`: persistent
3. `%300-%499`: persistent
4. `%500-`: save
5. `%p.*`, `%sys.*`, `%global.*`: persistent
6. `%v.*`, `%tmp.*`: volatile
7. その他の名前付きレジスタ: save

## 7. 実装方針

1. コア命令は直交性を維持する
2. 互換命令は段階的に外部ライブラリへ移す
3. 命令追加は `CommandRegistry` に登録し、Parserへ直書きしない
4. 仕様変更時はこの文書とコア仕様を同時更新する
