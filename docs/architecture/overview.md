# アーキテクチャ概要

このドキュメントでは、AriaEngineの全体アーキテクチャについて説明します。

## エンジンの全体構成

AriaEngineは以下の主要コンポーネントで構成されています：

```
AriaEngine
├── Core/              # コアシステム
│   ├── VirtualMachine.cs      # 仮想マシン
│   ├── Parser.cs              # パーサー
│   ├── OpCode.cs              # オペコード定義
│   ├── GameState.cs           # ゲーム状態管理
│   ├── Sprite.cs              # スプライトクラス
│   ├── ErrorReporter.cs       # エラー報告
│   └── Config.cs              # 設定管理
├── Rendering/         # レンダリングシステム
│   ├── SpriteRenderer.cs      # スプライトレンダラー
│   ├── TransitionManager.cs   # トランジション管理
│   └── TweenManager.cs        # Tweenアニメーション管理
├── Input/             # 入力システム
│   └── InputHandler.cs        # 入力処理
├── Audio/             # オーディオシステム
│   └── AudioManager.cs        # オーディオ管理
└── assets/            # ゲームアセット
    ├── fonts/                 # フォントファイル
    ├── bg/                    # 背景画像
    ├── ch/                    # キャラクター画像
    └── scripts/               # スクリプトファイル
```

## 主要クラスの役割

### VirtualMachine (仮想マシン)

**役割**: スクリプトの実行エンジン

**主な機能**:
- パースされた命令の実行
- プログラムカウンタの管理
- コールスタックの管理
- ゲーム状態の管理
- 制御フローの制御（ジャンプ、条件分岐、ループ）

**関連ファイル**: `Core/VirtualMachine.cs`

---

### Parser (パーサー)

**役割**: スクリプトファイルの解析

**主な機能**:
- テキストファイルのトークン化
- 命令の生成
- ラベルの解析
- 構文エラーの検出
- インライン構文の展開（`キャラクター「テキスト」`）

**関連ファイル**: `Core/Parser.cs`

---

### GameState (ゲーム状態)

**役割**: ゲームの状態管理

**主な機能**:
- レジスタの管理（%0-%9）
- 文字列変数の管理（$name）
- スプライトの管理
- フラグの管理
- カウンターの管理
- VM状態の管理（実行中、待機中など）

**関連ファイル**: `Core/GameState.cs`

---

### Sprite (スプライト)

**役割**: ビジュアル要素の表現

**主な機能**:
- 画像、テキスト、矩形の表示
- 位置、スケール、回転の管理
- 透明度の管理
- Zオーダーの管理
- 装飾効果（枠線、影、角丸）
- ボタンとしての機能
- ホバーエフェクト

**関連ファイル**: `Core/Sprite.cs`

---

### SpriteRenderer (スプライトレンダラー)

**役割**: スプライトの描画

**主な機能**:
- スプライトのZオーダーソート
- テキストの描画とラッピング
- フォントのロードと管理
- 色の解析と適用
- 地震エフェクトの適用
- トランジションエフェクトの適用

**関連ファイル**: `Rendering/SpriteRenderer.cs`

---

### TweenManager (Tweenマネージャー)

**役割**: アニメーションの管理

**主な機能**:
- Tweenアニメーションの登録
- アニメーションの更新
- イージング関数の適用
- アニメーションの完了検知

**関連ファイル**: `Rendering/TweenManager.cs`

---

### InputHandler (入力ハンドラー)

**役割**: ユーザー入力の処理

**主な機能**:
- マウス入力の検知
- キーボード入力の検知
- ボタンのクリック検知
- 右クリックメニューの表示
- デバッグモードのトグル（F3）

**関連ファイル**: `Input/InputHandler.cs`

---

### AudioManager (オーディオマネージャー)

**役割**: オーディオの再生と管理

**主な機能**:
- BGMの再生と停止
- SEの再生
- 音量の制御
- フェードイン/フェードアウト

**関連ファイル**: `Audio/AudioManager.cs`

---

## データフロー

### スクリプト実行のフロー

```
1. スクリプトファイル (.aria)
   ↓
2. Parserによる解析
   ↓
3. 命令リスト (List<Instruction>)
   ↓
4. VirtualMachineによる実行
   ↓
5. GameStateの更新
   ↓
6. SpriteRendererによる描画
   ↓
7. 画面に表示
```

### メインゲームループ

```
while (!WindowShouldClose):
    1. Update(deltaTime)
       - VMの状態更新
       - 入力処理
       - アニメーション更新
       - オーディオ更新
       - トランジション更新

    2. Step VM (Running状態の場合)
       - 次の命令を実行

    3. Render()
       - 画面クリア
       - スプライト描画（Zオーダー順）
       - トランジション描画
       - デバッグ情報描画
```

## スクリプト言語の処理

### パース処理

1. **ファイル読み込み**: `.aria`ファイルを行単位で読み込み
2. **コメント除去**: `;`で始まる行をコメントとして除去
3. **トークン化**: 行をトークンに分割
4. **命令生成**: トークンを`Instruction`オブジェクトに変換
5. **ラブル解決**: ラベルのアドレスを解決

### 実行処理

1. **命令フェッチ**: プログラムカウンタから命令を取得
2. **オペコード解析**: 命令の種類を特定
3. **引数評価**: 引数を評価
4. **命令実行**: オペコードに対応する処理を実行
5. **状態更新**: ゲーム状態を更新
6. **プログラムカウンタ更新**: 次の命令へ

## スプライトシステム

### スプライトの種類

1. **Imageスプライト**: 画像を表示
2. **Textスプライト**: テキストを表示
3. **Rectスプライト**: 矩形を表示

### スプライトのライフサイクル

```
1. 作成 (lsp, lsp_text, lsp_rect)
   ↓
2. プロパティ設定 (sp_alpha, sp_scaleなど)
   ↓
3. 表示 (vsp on)
   ↓
4. アニメーション (amsp, afadeなど)
   ↓
5. 削除 (csp)
```

## セーブ/ロードシステム

### セーブデータの構造

```csharp
public class SaveData
{
    public Dictionary<int, int> Registers { get; set; }        // %0-%9
    public Dictionary<string, string> StringRegisters { get; set; } // $name
    public Dictionary<string, bool> Flags { get; set; }       // フラグ
    public Dictionary<string, int> Counters { get; set; }     // カウンター
    public List<SpriteData> Sprites { get; set; }             // スプライト状態
    public int ProgramCounter { get; set; }                   // プログラムカウンタ
    public string CurrentLabel { get; set; }                  // 現在のラベル
    // ... その他の状態
}
```

### セーブ/ロードのフロー

```
セーブ:
1. GameStateから現在の状態を取得
2. SaveDataオブジェクトにシリアライズ
3. JSONファイルに保存

ロード:
1. JSONファイルから読み込み
2. SaveDataオブジェクトにデシリアライズ
3. GameStateに状態を復元
4. VMを復元した状態から再開
```

## パフォーマンスの考慮事項

### レンダリング

- **Zオーダーソート**: 描画前にスプライトをZオーダーでソート
- **フォントアトラス**: フォントをテクスチャアトラスにキャッシュ
- **バッチ描画**: 可能な限りバッチ描画を使用

### メモリ管理

- **スプライトプール**: 頻繁に使用するスプライトはプール化
- **テクスチャキャッシュ**: テクスチャをキャッシュして再利用
- **ガベージコレクション**: 不必要なオブジェクトを適切に解放

### スクリプト実行

- **ジャンプテーブル**: ラベルジャンプをハッシュテーブルで高速化
- **命令キャッシュ**: 頻繁に実行する命令をキャッシュ
- **レイジー評価**: 必要な時まで評価を遅延

## 拡張性

### 新しいオペコードの追加

1. `OpCode.cs`に新しいオペコードを追加
2. `Parser.cs`にコマンドマッピングを追加
3. `VirtualMachine.cs`に実装を追加

### 新しいスプライトタイプの追加

1. `Sprite.cs`に新しいタイプを追加
2. `SpriteRenderer.cs`に描画ロジックを追加
3. パーサーに新しいコマンドを追加

### 新しいイージング関数の追加

1. `TweenManager.cs`に関数を追加
2. イージング名をマッピングに追加

## まとめ

AriaEngineのアーキテクチャは以下の特徴を持っています：

1. **モジュール化**: 各コンポーネントが明確に分離されている
2. **スクリプト主導**: ゲームロジックはスクリプトで記述
3. **拡張性**: 新しい機能を簡単に追加可能
4. **パフォーマンス**: 効率的なレンダリングとメモリ管理
5. **保守性**: コードが整理され、理解しやすい
