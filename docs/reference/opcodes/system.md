## セーブ・ロード

### `saveon`

セーブ機能を有効にします。

**カテゴリ**: Save

```aria
saveon
```

`saveon` を実行後、`save` コマンドでセーブデータを作成できます。初期状態では有効です。

**例**:
```aria
saveon
save 1
```

---

### `saveoff`

セーブ機能を無効にします。

**カテゴリ**: Save

```aria
saveoff
```

`saveoff` 状態では `save` コマンドは実行されません。

**例**:
```aria
saveoff
save 1          ; 無効化されているためセーブされない
```

---

### `save`

指定したスロット番号にゲームをセーブします。

**カテゴリ**: Save

```aria
save スロット番号
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `スロット番号` | 整数 | セーブスロット番号（0以上） |

セーブ時に画面のサムネイル（320x180）が自動生成され、セーブデータに含まれます。セーブデータには現在のスクリプトファイル名、プログラムカウンタ、レジスタ、フラグ、スプライト状態などが含まれます。

**例**:
```aria
save 1          ; スロット1にセーブ
save %slot      ; レジスタでスロット番号を指定
```

**注意**:
- `saveon` が有効な場合のみ実際にセーブされます
- スロット0はオートセーブ専用です（`autosave` コマンドで使用）

---

### `load`

指定したスロット番号からゲームをロードします。

**カテゴリ**: Save

```aria
load スロット番号
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `スロット番号` | 整数 | ロードするセーブスロット番号 |

ロード後、スクリプトの `*load_restore` ラベルが存在する場合はそこにジャンプします。存在しない場合は、セーブ時の状態でスクリプト実行を再開します。

**例**:
```aria
load 1          ; スロット1からロード
load %slot      ; レジスタでスロット番号を指定
```

---

### `saveinfo`

指定したスロットのセーブ情報を取得します。

**カテゴリ**: Save

```aria
saveinfo スロット番号, $プレビュー変数, $日時変数, %存在フラグ
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `スロット番号` | 整数 | 調べるセーブスロット番号 |
| `$プレビュー変数` | 文字列レジスタ | セーブ時のテキストプレビューを受け取る変数 |
| `$日時変数` | 文字列レジスタ | セーブ日時（`yyyy/MM/dd HH:mm`形式）を受け取る変数 |
| `%存在フラグ` | 整数レジスタ | セーブデータが存在すれば1、なければ0が設定される |

**例**:
```aria
saveinfo 1, $preview, $datetime, %exists
if %exists == 1
    text "[$datetime] $preview"
endif
```

## ゲームフロー制御

### `end`

ゲームを終了します。

**カテゴリ**: System

```aria
end
```

VMの状態を `Ended` に設定し、ウィンドウを閉じるリクエストを発行します。

**例**:
```aria
*title
    text "ゲームを終了しますか？"
    yesnobox %result, "終了しますか？", 1
    if %result == 1
        end
    endif
```

## ウィンドウ・環境設定

### `window`

ウィンドウサイズとタイトルを設定します。

**カテゴリ**: System

```aria
window 幅, 高さ, "タイトル"
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `幅` | 整数 | ウィンドウ幅（ピクセル） |
| `高さ` | 整数 | ウィンドウ高さ（ピクセル） |
| `タイトル` | 文字列 | ウィンドウタイトル |

**注意**:
- `init.aria` で実行された場合、エンジン起動時のウィンドウサイズとして適用されます
- 実行時に呼び出しても、ウィンドウサイズは変更されません（`setconfig fullscreen` を使用してください）
- タイトルは `caption` / `window_title` で後から変更可能です

**例**:
```aria
window 1280, 720, "My Visual Novel"
```

---

### `caption`

ウィンドウタイトルを設定します。

**カテゴリ**: System

```aria
caption "タイトル"
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `タイトル` | 文字列 | 新しいウィンドウタイトル |

`window_title` と同じ機能です。文字列補間 `${...}` には対応していません（リテラル文字列として扱われます）。

**例**:
```aria
caption "Chapter 1 - 始まり"
```

---

### `window_title`

ウィンドウタイトルを設定します。

**カテゴリ**: System

```aria
window_title "タイトル"
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `タイトル` | 文字列 | 新しいウィンドウタイトル |

`caption` と同じ機能です。`GetString` を通すため、文字列補間 `${...}` に対応しています。

**例**:
```aria
window_title "${$chapter_name}"
```

---

### `debug`

デバッグモードを切り替えます。

**カテゴリ**: System

```aria
debug on
```

```aria
debug off
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `on/off` | 文字列 | `on` で有効、`off` で無効（デフォルトは無効） |

デバッグモードが有効な場合、画面左上にFPS、プログラムカウンタ、スプライト数、ボタン当たり判定が表示されます。F3キーでも実行時に切り替え可能です。

**例**:
```aria
debug on
```

## システムUI

### `systemcall`

システムメニューを開きます。

**カテゴリ**: System

```aria
systemcall "コマンド名"
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `コマンド名` | 文字列 | 実行するシステムコマンド |

使用可能なコマンド名:

| コマンド名 | 動作 |
|-----------|------|
| `rmenu` | 右クリックメニュー（メインメニュー）を開く |
| `autosave` | オートセーブを実行 |
| `autoload` / `load_auto` | オートセーブデータをロード |
| `lookback` | バックログ（テキスト履歴）を開く |
| `load` | ロード画面を開く |

**例**:
```aria
systemcall "rmenu"        ; メインメニューを開く
systemcall "autosave"     ; オートセーブ
systemcall "lookback"     ; バックログ表示
```

### `system_button`

システムボタンの表示・非表示を切り替えます。

**カテゴリ**: System

```aria
system_button "ボタン名", on/off
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `ボタン名` | 文字列 | `close`/`end`, `reset`, `skip`, `save`, `load` のいずれか |
| `on/off` | 文字列/整数 | `on`/`1`/`true`/`yes` で表示、`off`/`0`/`false`/`no` で非表示 |

**例**:
```aria
system_button "save", on
system_button "load", on
system_button "close", off
```

### `yesnobox`

Yes/No選択ダイアログを表示します。

**カテゴリ**: System

```aria
yesnobox %結果レジスタ, "メッセージ", デフォルト選択
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `%結果レジスタ` | 整数レジスタ | 選択結果を受け取るレジスタ（1=Yes, 0=No） |
| `メッセージ` | 文字列 | ダイアログに表示するメッセージ |
| `デフォルト選択` | 整数 | デフォルトの選択（現在未使用） |

`compat_mode on` が必要です。`compat_mode off` では警告が出力され、UIは生成されません。自動的に `btnwait` 相当の待機状態になります。

**例**:
```aria
yesnobox %result, "ゲームを終了しますか？", 1
if %result == 1
    end
endif
```

---

### `mesbox`

OKボタンのみのメッセージダイアログを表示します。

**カテゴリ**: System

```aria
mesbox "メッセージ", "タイトル"
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `メッセージ` | 文字列 | ダイアログ本文 |
| `タイトル` | 文字列 | ダイアログタイトル |

`compat_mode on` が必要です。自動的に `btnwait` 相当の待機状態になり、結果は `%0` に設定されます（常に1）。

**例**:
```aria
mesbox "セーブが完了しました。", "確認"
```

## スコープ管理

### `scope`

新しいローカルスコープを開始します。

**カテゴリ**: System

```aria
scope
```

スコープ内で作成された変数（整数・文字列）とスプライトは、スコープ終了時に自動的に破棄されます。`defer` で登録した命令はスコープ終了時にLIFO順で実行されます。

スコープはネスト可能です。`goto`/`break`/`continue`/`return` でスコープを抜ける際も、自動的にクリーンアップが実行されます。

**例**:
```aria
scope
    let %local_var = 100
    lsp 50, "temp.png", 0, 0
    defer csp 50          ; スコープ終了時に自動実行
end_scope
; %local_var とスプライト50は破棄される
```

---

### `end_scope`

すべてのスコープを終了し、クリーンアップを実行します。

**カテゴリ**: System

```aria
end_scope
```

スコープスタックが空になるまで、各スコープの `defer` 命令を実行し、ローカル変数とスコープ内スプライトを破棄します。

**例**:
```aria
scope
    let %x = 10
end_scope
; %x は破棄される
```

**注意**:
- 通常は `goto`/`break`/`continue`/`return` などの制御フローが自動的にスコープをクリーンアップします
- `end_scope` を明示的に呼ぶと、手動でスコープを閉じることができます

## バックログ

### `backlog_count`

バックログ（テキスト履歴）のエントリ数を取得します。

**カテゴリ**: System

```aria
backlog_count %結果レジスタ
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `%結果レジスタ` | 整数レジスタ | バックログエントリ数を受け取るレジスタ |

バックログは `wait`/`@`/`|` などのクリック待機時に自動的に記録されます。最大300件まで保持され、それを超えると古いものから削除されます。

**例**:
```aria
backlog_count %count
text "履歴数: ${%count}"
```

---

### `backlog_entry`

指定したインデックスのバックログエントリを取得します。

**カテゴリ**: System

```aria
backlog_entry インデックス, $テキスト変数
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `インデックス` | 整数 | 取得するエントリのインデックス（0から始まる） |
| `$テキスト変数` | 文字列レジスタ | バックログテキストを受け取る変数 |

テキスト内の改行は ` / ` に置き換えられて返されます。範囲外のインデックスを指定すると空文字列が返されます。

**例**:
```aria
backlog_count %total
if %total > 0
    backlog_entry %total - 1, $latest
    text "最新: $latest"
endif
```

### `gallery_info`

指定したインデックスのギャラリーエントリ情報を取得します。

**カテゴリ**: System

```aria

```

| 引数 | 型 | 説明 |
|------|-----|------|
| `インデックス` | 整数 | エントリのインデックス（0から始まる） |
| `$タイトル変数` | 文字列レジスタ | タイトルを受け取る変数 |
| `$パス変数` | 文字列レジスタ | 画像パスを受け取る変数 |

```aria

    text "[$title] - 閲覧可能"
endif
```

## コンフィグ設定

### `getconfig`

現在の設定値を取得します。

**カテゴリ**: System

```aria
getconfig %結果レジスタ, "設定キー"
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `%結果レジスタ` | 整数レジスタ | 設定値を受け取るレジスタ |
| `設定キー` | 文字列 | 取得する設定のキー名 |

対応する設定キー（大文字小文字不敏感、`_` は無視されます）:

| キー名 | 取得する値 |
|--------|-----------|
| `bgmvol` / `bgmvolume` | BGM音量（0〜100） |
| `sevol` / `sevolume` | SE音量（0〜100） |
| `textspeed` / `textspeedms` | テキスト表示速度（ミリ秒） |
| `skipunread` / `skipunreadmode` | 未読スキップ設定（1=有効, 0=無効） |
| `backlog` / `backlogenabled` | バックログ有効（1=有効, 0=無効） |
| `clickcursor` / `showclickcursor` | クリックカーソル表示（1=有効, 0=無効） |
| `autowait` / `automodewait` / `automodewaittimems` | オートモード待機時間（ミリ秒） |
| `fullscreen` | フルスクリーン（1=有効, 0=無効） |
| その他 | 0 |

**例**:
```aria
getconfig %vol, "bgmvol"
text "BGM音量: ${%vol}"
```

---

### `setconfig`

設定値を変更します。

**カテゴリ**: System

```aria
setconfig "設定キー", 値
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `設定キー` | 文字列 | 変更する設定のキー名 |
| `値` | 整数 | 新しい設定値 |

対応する設定キーと値の範囲:

| キー名 | 値の範囲 | 効果 |
|--------|---------|------|
| `bgmvol` / `bgmvolume` | 0〜100 | BGM音量 |
| `sevol` / `sevolume` | 0〜100 | SE音量 |
| `textspeed` / `textspeedms` | 0以上 | テキスト表示速度（0で即時表示） |
| `skipunread` / `skipunreadmode` | 0/1 | 未読スキップ設定 |
| `backlog` / `backlogenabled` | 0/1 | バックログ有効/無効 |
| `clickcursor` / `showclickcursor` | 0/1 | クリックカーソル表示 |
| `autowait` / `automodewait` / `automodewaittimems` | 100〜10000 | オートモード待機時間（ミリ秒） |
| `fullscreen` | 0/1 | フルスクリーン切り替え |

**例**:
```aria
setconfig "bgmvol", 80
setconfig "textspeed", 20
setconfig "fullscreen", 1
```

**注意**:
- `textspeed` を変更すると `config.json` の `GlobalTextSpeedMs` も更新されます
- `fullscreen` を変更すると、即座に画面モードが切り替わります
- 変更内容は永続データとしてマークされますが、`saveconfig` を実行するまでファイルには保存されません

---

### `saveconfig`

現在の設定を `config.json` に保存します。

**カテゴリ**: System

```aria
saveconfig
```

`setconfig` で変更した値や、レジスタ・フラグなどの永続データを含めて `config.json` に書き出します。オートセーブのタイミング（1秒間隔）でも自動的に保存されます。

**例**:
```aria
setconfig "bgmvol", 50
saveconfig
```
